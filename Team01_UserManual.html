1. Introduction

TaskFlow is a web-based personal task management application. It allows you to create, organise, and track your tasks using categories, priorities, and statuses — all from any modern web browser.

Key features:

  • Secure account registration and login
  • Create, edit, and delete tasks with titles, descriptions, priorities, and due dates
  • Organise tasks with custom categories
  • Filter tasks by status, priority, or category
  • Search tasks by keyword
  • One-click toggle to mark tasks as done

3. Deployment Instructions

3.1 Install Node.js

1
Download the Node.js LTS installer from nodejs.org and run it. Verify the installation by opening a terminal and running:
node --version and npm --version

3.2 Get the source code

2
Copy the Team01_Project folder to the server (e.g. C:\TaskFlow or /opt/taskflow).

3.3 Install dependencies

3
Open a terminal, navigate to the backend folder, and install dependencies: 
cd Team01_Project/backend
npm install
Note: The better-sqlite3 package compiles a native module. On Windows ensure you have the Windows Build Tools installed (npm install -g windows-build-tools in an administrator PowerShell).

3.4 Configure environment (optional)

4
You may set these environment variables before starting the server:
VariableDefaultDescription
PORT3000TCP port the server listens on
JWT_SECRETtaskflow_secret_2024Secret key used to sign JWT tokens — change this in production

3.5 Start the server

5
Start the server with: 
node server.js
You should see: TaskFlow server running on http://localhost:3000
6
Open your browser and navigate to http://localhost:3000 (or the server's IP address and port). The TaskFlow login page should appear.

3.6 Run automated tests

npm test

Jest will run all test suites and print a coverage report. All tests should pass before deployment.

3.7 Running in production (optional)

For a persistent server process, use PM2:

npm install -g pm2
pm2 start server.js --name taskflow
pm2 startup    # auto-start on reboot
pm2 save
Warning: Change the JWT_SECRET environment variable to a long random string before exposing the application to the internet.

4. Getting Started — Creating an Account

1
Open the application in your browser. The Login page appears.
2
Click the "Create one" link at the bottom of the form. You are taken to the Register page.
3
Fill in the three fields:
  • Username — a unique display name (e.g. johndoe)
  • Email address — your email (e.g. john@example.com)
  • Password — at least 6 characters
4
Click "Create Account". If all fields are valid you are automatically logged in and taken to the Dashboard.
If you see a red error banner, read the message: it will tell you if your email is already in use, your username is taken, or your password is too short.

5. Signing In and Out

Sign in

1
On the Login page enter your email and password.
2
Click "Sign In". You are redirected to your Dashboard.

Sign out

1
In the top-right corner of any page, click the "Logout" button. Your session is cleared and you are returned to the Login page.

6. Managing Tasks

6.1 Create a new task

1
On the Dashboard, click the blue "+ New Task" button in the top-right of the content area. The task modal opens.
2
Enter a Title (required). Optionally add:
  • Description — extra details about the task
  • Status — To Do / In Progress / Done (default: To Do)
  • Priority — Low / Medium / High (default: Medium)
  • Due Date — a date by which the task should be completed
  • Category — group the task under a category you have created
3
Click "Save Task". The modal closes and the new task appears at the top of the task list.

6.2 Edit a task

1
Click anywhere on a task card (not the checkbox). The task modal opens with the current values pre-filled.
2
Modify any field and click "Save Task".

6.3 Mark a task as done (quick toggle)

1
Click the circular checkbox on the left side of the task card. The task is immediately marked as Done (title is struck through). Click again to return to To Do.

6.4 Delete a task

1
Open the task by clicking its card.
2
Click the red "Delete" button (bottom-left of the modal).
3
Confirm the deletion in the browser dialog. The task is permanently removed.

7. Filtering and Searching Tasks

Filter by status

In the left sidebar, open the Status dropdown and select To Do, In Progress, or Done. The task list updates immediately. Select All to remove the filter.

Filter by priority

Use the Priority dropdown in the sidebar to show only tasks of a given priority level.

Filter by category

Use the Category dropdown to show only tasks belonging to a specific category.

Search by keyword

Type a word or phrase in the Search tasks… box at the top of the content area. The list filters automatically as you type (with a short 300 ms delay). Clear the box to show all tasks again.

You can combine multiple filters simultaneously. For example, you can filter for High priority + In Progress tasks at the same time.

8. Managing Categories

8.1 Create a category

1
In the left sidebar, scroll to the Categories section.
2
Type the category name in the input field (e.g. Work, Personal, Study).
3
Click "Add". The new category appears in the sidebar list and in all category dropdowns.

8.2 Assign a category to a task

1
Open the New Task or Edit Task modal.
2
In the Category dropdown, select the desired category.
3
Save the task. The category badge appears on the task card.

8.3 Delete a category

1
In the sidebar, click the × button next to the category name.
2
The category is removed. Tasks previously assigned to it remain, but their category is cleared.
Warning: Category deletion is permanent and cannot be undone.

9. Troubleshooting

ProblemLikely causeSolution
Page shows "Loading tasks…" forever Server not running or wrong URL Check that the Node.js server is running on port 3000 and visit http://localhost:3000
"Invalid or expired token" error JWT token expired after 24 h Log out and log in again
Registration error: "Username or email already exists" Duplicate credentials Use a different username or email address
Task not saving, modal stays open Title field is empty The title field is required — fill it in and try again
Cannot install better-sqlite3 on Windows Missing native build tools Run npm install -g windows-build-tools as administrator, then npm install again
Server exits immediately after start Port 3000 already in use Set PORT=3001 environment variable, or stop the process occupying port 3000
Tests fail with "cannot find module" Dependencies not installed Run npm install inside the backend folder
