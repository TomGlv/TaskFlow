Application Workflow Overview

The diagram below shows the high-level navigation flow between all application screens.

Landing / Login
→
Dashboard
→
New Task Modal
Register Page
→
Dashboard
→
Edit Task Modal

Unauthenticated visitors land on the Login page. Successful login/registration redirects to the Dashboard. From the Dashboard, users access New/Edit Task modals. Logout returns to the Login page.

Elements

  • Logo + brand name — top of the card
  • Email input — type="email", required
  • Password input — type="password", required
  • Sign In button — full-width primary button
  • Register link — navigates to Register page
  • Error banner — shown on invalid credentials (hidden by default)

Behaviour

  • Form submits via JavaScript (fetch POST /api/auth/login)
  • On success → JWT stored in localStorage, redirect to Dashboard
  • On error → error banner displayed with message
  • Already authenticated users are redirected immediately

  • Username input — text, required, unique
  • Email input — type="email", required, unique
  • Password input — minlength=6
  • Create Account button — full-width primary
  • Login link — back to Login page

Behaviour

  • Calls POST /api/auth/register
  • On success → JWT stored, redirect to Dashboard
  • Duplicate email/username → 409 error shown in banner
  • Short password → 400 error shown in banner

Sidebar elements

  • Status dropdown filter
  • Priority dropdown filter
  • Category dropdown filter
  • Category list with delete buttons
  • New category input + Add button

Content area elements

  • Search box (live, debounced 300 ms)
  • New Task button → opens modal
  • Statistics row: total / to-do / in-progress / done
  • Task cards (click to edit, checkbox to toggle done)
  • Left border colour = priority indicator

Fields

  • Title — required text input
  • Description — optional textarea (3 rows)
  • Status — select: To Do / In Progress / Done
  • Priority — select: Low / Medium / High
  • Due Date — optional date picker
  • Category — select: None or any user category

Actions

  • Save — creates or updates task, closes modal
  • Cancel / × — closes modal without saving
  • Delete — visible only in edit mode; confirms then deletes
  • Backdrop click — closes modal without saving

Workflow Description

Primary Flow — Create and complete a task

1. Login
→
2. Dashboard
→
3. Click "+ New Task"
→
4. Fill form, Save
→
5. Task appears in list
→
6. Click checkbox → Done

Alternate Flow — Edit an existing task

Dashboard
→
Click task card
→
Edit Task Modal opens with data
→
Modify fields, Save
→
Task list refreshed

Alternate Flow — Filter tasks

Dashboard
→
Select Status / Priority / Category in sidebar
→
Task list filters automatically

Alternate Flow — Manage categories

Sidebar → type category name → Add
→
Category appears in sidebar and dropdowns
Sidebar → click × next to category
→
Category deleted, tasks cleared (no cascade delete of tasks)
